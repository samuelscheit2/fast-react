# worker-065-dom-event-delegation-oracle

## Objective

Add deterministic React DOM 19.2.6 delegated event registration and dispatch behavior oracle files.

Write scope honored: only `tests/conformance/src/dom-event-delegation-*.mjs`, `tests/conformance/scripts/*dom-event-delegation*.mjs`, `tests/conformance/test/dom-event-delegation-oracle.test.mjs`, `tests/conformance/oracles/react-19.2.6-dom-event-delegation-oracle.json`, and this report were changed.

## Summary

The oracle records React DOM 19.2.6 delegated event behavior from exact pinned `react`, `react-dom`, and `scheduler` tarballs referenced by the checked runtime inventory. It uses an isolated temporary `node_modules` tree, verifies tarball integrity and file lists against the inventory, installs a deterministic minimal DOM host, and probes each event scenario in a separate Node child process for both development and production default Node modes.

Covered behavior:

- `createRoot` listener installation evidence on the root container, including 138 root-container listener registrations in both development and production.
- Owner-document `selectionchange` listener installation and React listening marker evidence.
- Passive listener option evidence for delegated `wheel` listeners.
- Capture and bubble dispatch order for delegated `click` and `mousemove`.
- `stopPropagation` skipping the ancestor bubble listener.
- `preventDefault` updating synthetic and native `defaultPrevented` state and `dispatchEvent` return value.
- Synthetic event shape for selected mouse and wheel fields, `persist`, default/prevent/propagation helpers, target/currentTarget values during dispatch, and retained synthetic event `currentTarget` reset after dispatch.
- Selected discrete and continuous event examples: `click`, `mousemove`, and `wheel`.

Intentional gaps:

- No Fast React DOM behavior comparison is claimed; this is a React DOM 19.2.6 behavior oracle only.
- Event priority lane claims are excluded because worker 048 owns that oracle surface.
- Browser matrix, hydration replay, portal retargeting, non-delegated scroll/media events, and form/change/select/beforeinput plugin behavior are separate surfaces.

## Evidence Gathered

- The checked oracle regenerated byte-for-byte from the generator.
- The targeted test asserts schema, targets, conformance claims, listener installation, capture/bubble order, `stopPropagation`, `preventDefault`, synthetic event shape, continuous examples, print CLI behavior, and artifact path leak guards.
- Scoped searches found no local temporary path leaks in the changed conformance files and no event-priority lane assertions beyond explicit out-of-scope guards.
- Two read-only explorer subagents were spawned to test hypotheses:
  - One reviewed conformance-pattern fit, write scope, deterministic generation, and path-leak checks.

  Their findings were used as review inputs; direct local verification remained the source of truth.

## Quality Review

Maintainability: the implementation follows existing conformance oracle structure: target constants, scenario list, probe runner, generator, print CLI, checked artifact, and a focused `node:test` suite. Scenario definitions keep event cases declarative while the probe runner owns host behavior.

Performance: generator work is intentionally isolated and bounded by child-process timeouts. Runtime tests read the checked artifact and avoid network access; only explicit regeneration downloads tarballs.

Security: package lifecycle scripts are not run. Tarball integrity and file lists are verified before probing. The generated artifact omits temporary extraction paths, randomized React listening marker names, callback identities, and concrete local paths.

## Risks Or Blockers

- The deterministic minimal DOM host is not a browser conformance matrix. It is sufficient for delegated listener installation and selected dispatch behavior, but later browser-backed or jsdom-backed oracles should cover focus, selection, IME, form, and layout-sensitive cases.
- Listener counts are intentionally exact for React DOM 19.2.6. They should be regenerated, not hand-edited, if the pinned React DOM version changes.
- Fast React compatibility remains unclaimed until a DOM event implementation exists and can be dual-run against this oracle.

## Recommended Next Tasks

- Use this oracle to guide the first React DOM delegated listener registry and synthetic dispatch queue implementation.
- Add separate oracles for portal event retargeting, hydration replay, non-delegated events, and form/change/select/beforeinput plugins.
- Keep event priority lane behavior in the worker 048 oracle surface rather than mixing it into delegated dispatch checks.
